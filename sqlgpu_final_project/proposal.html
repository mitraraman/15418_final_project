Project Proposal:
SQL on GPU with Efficient Relational Algebra Algorithms
Yingchao Liu (yingchal) and Mitra Raman (mitrar)
Main Project Page
Summary
We plan on implementing the database operations PROJECT, SELECT, and JOIN by using relational algebra on GPU machines in order to investigate the viability of accelerating a database on GPU. We will use memsql on a multi-core CPU as reference.
Background
As all of us know, the traditional database performance is bad (since access to disk is really slow), so here comes Memsql. Memsql places data into memory and translates SQL into C++ for the utmost optimization in query execution, which enables Memsql to write and read data at incredible speeds.

But we might not satisfy the speed of the limited number of CPU cores provide to us. For certain types of queries, tuples are highly independent each other, which has high potential to benefit from parallelism. Specifically, if we can have one thread to work on each tuple simultaneously, the performance of large volume of data could be largely promoted.

So our project is to push the bar even higher. We like in-memory database, and we also want an even faster way to ingest large volumes of data, so we would try to put the data into GPUâ€™s memory and utilize the huge computation power of GPU.
Challenge
Utilize the memory hierarchy of GPU to make memory access more efficient in GPU ( global , shared, register )

One possible challenge is the potential of divergence. For example, a tuple might have many attributes. Some attributes of some certain tuples might satisfy a user-defined predicate while other do not. In this case, divergence appears, and some threads could be idle for a long time while a few threads are super busy working on the qualified tuples.

We want to learn what types of query makes GPU-based database viable
